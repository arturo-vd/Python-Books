## How to Contribute

### 1. Adding Resources
- Books, videos, tutorials, articles
- Make sure they're high-quality and relevant
- Add proper categorization

### 2. Improving Documentation
- Fix typos and grammatical errors
- Clarify confusing sections
- Add more examples

### 3. Adding Projects
- Include detailed README
- Provide starter code
- Add solution/reference implementation

### 4. Fixing Issues
- Check existing issues
- Comment before starting work
- Submit PR with fixes

## Guidelines

### File Naming
- Use descriptive names
- Use hyphens for spaces
- Include version/date if relevant
- Example: `python-crash-course-v2.pdf`

### Folder Organization
- Follow existing structure
- Don't create new top-level folders without discussion
- Keep related files together

### Quality Standards
- Resources should be beginner-friendly
- Include sources/links
- Verify all external links work
- Test all code before submitting

## Pull Request Process

1. Fork the repository
2. Create feature branch (`git checkout -b feature/amazing-feature`)
3. Commit changes (`git commit -m 'Add amazing feature'`)
4. Push to branch (`git push origin feature/amazing-feature`)
5. Open Pull Request

### PR Title Format
- `[Category] Brief description`
- Examples:
  - `[Python] Add advanced OOP examples`
  - `[Frontend] Update React tutorial links`
  - `[Docs] Fix typos in README`

### PR Description Should Include
- What was added/changed
- Why it's useful
- Any relevant issues it fixes

## Code of Conduct

### Our Standards
- Be respectful and inclusive
- Provide constructive feedback
- Help others learn
- Give credit where due

### Unacceptable Behavior
- Harassment or discrimination
- Spam or self-promotion
- Plagiarism
- Sharing copyrighted content without permission

## Questions?

Open an issue or discussion if you:
- Need clarification
- Want to propose major changes
- Have suggestions for improvement
